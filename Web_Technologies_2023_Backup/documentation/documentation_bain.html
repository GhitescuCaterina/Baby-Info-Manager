1.4 Product Scope

The BaIn (Baby Info Manager) web application allows authenticated users, such as families or couples, to manage resources related to the care of a child from infancy to puberty. The application offers features such as a calendar for feeding and sleeping hours, multimedia resources like photos, films, and audio recordings that capture the child's growth and development, medical history, and records of their relationship with other children, such as cousins or kindergarten/school colleagues.

The application provides a chronological view of the most important captured moments related to each individual child. This timeline feature allows parents or guardians to easily track and review their child's milestones and growth over time.

This web application is designed to improve the experience of caring for children by allowing users to have all the resources and information related to childcare in one organized and easily accessible place. In addition, the application provides an easy way to share this information with other family members or close friends by either tagging them in various multimedia resources, or by sharing important moments through a social application or an RSS news feed.

The goal of this application is to provide a complete solution for managing resources related to childcare, so that users can be more organized and efficient in managing their childcare responsibilities. This application can be a useful tool for parents, grandparents, tutors, or anyone else who has childcare responsibilities.

2.2 Product Functions

The BaIn web application is designed to allow authenticated users to manage resources related to the care of a child from infancy to puberty. The major functions of the application include:

  • The application allows users to create a parent/tutor account with some basic personal information regarding their e-mail, last name, first name, date of birth and password (the latter must be confirmed by introducing it twice).
  • Any user that has an account can log in to the application using their e-mail and password.
  • For every account created, a unique reference code is generated, which can be used to invite other users to acces the data regarding the initial user's children.
  • Any user that has an account can view a list of all the users that they have invited to view their children's profiles.
  • Any user that has an account can delete it, as a consequence, all the data related to their children will be deleted too.
  • Any user that is logged in can create profiles for their children, which will be stored in the database.
  • When creating the profile of a child, a user needs to provide a picture of the child, the first and last name of the child, a username (which is unique), the date of birth and the gender that their child has.
  • Any user that has created the profile of one or more children can view, edit or delete those specific profiles.
  • Any user that has an account can view a list of all the children that they have access to, which can be either their own children or the children of other users that have invited them to view their children's profiles.
  • Any user that has access to the profile of a child, can view or edit that profile.
  • Any user that has access to the profile of a child can add multimedia resources (photos, videos, audio recordings) to the profiles of those children and various information related to the children's medical records, memories, and feeding and sleeping times.
  • Any user that has access to the profile of a child can view, edit or delete the multimedia resources and information related to those children.
  • Any user that has access to the profile of a child can view a chronological timeline of the most important captured moments related to those children.
  • Any user that has an account and has access to the profile of a child can refer to another person's child by tagging them (using their username) when adding multimedia resources or other information to the profile of a child (when doing so, that piece of media will be visible on both the profiles of the children - the one that was tagged and the one that tagged them).
  • Any user that has access to the profile of a child can export the multimedia resources and information related to that child in a CSV or JSON file.
  • Any user that is logged in can create the profile of a child by just importing a CSV or JSON file that contains the required information.
  • Any user that has access to the profile of a child can share the most important moments related to that child through a social application or an RSS news feed.

2.3 User Classes and Characteristics

The two main user classes of the BaIn web application relate mainly to how one person has access to the account of a child, therefore there are the parents (the people that create the account of a child) and the tutors (the people that can have access to a child's account).

The parents are the ones that have the most privileges, as they can create, edit and delete the profiles of their children. The parents can invite other users to view the profiles of their children and they can see a list of people that have access to seeing their children’s profiles. The parents can also create the profiles of their children by just importing a CSV or JSON file that contains the required information.

The tutors are the ones that are invited to view the profiles of the children of other users. The tutors can modify the profiles of the children they have access to, but they can not delete them.

Both the parents and the tutors can create a personal account, log in to the application and delete their accounts. They can also export the multimedia resources and information related to their children in a CSV or JSON file, or share the most important moments related to their children/the children they have access to through a social application or an RSS news feed. Both types of users can tag the profiles of other users' children when adding multimedia resources or other information to the profiles of the children. They have access to a list of all the children’s profiles that they created/are alloowed to see. They can also view a chronological timeline of the most important captured moments related to the children they have access to.

2.4 Operating Environment

This application is designed to be used on a web browser, so it can be accessed from any device that has an internet connection and a web browser installed. The application is designed to be used on a desktop computer, but it can also be used on a mobile device, as it is responsive and it can be used on any screen size.

2.7 Assumptions and Dependencies

In order to function at its maximum potential, the BaIn web application needs to be connected to the internet, as it uses an external database to store the information related to the profiles of the children. The application also needs to be connected to the internet in order to be able to share the most important moments related to the children through a social application or an RSS news feed. The application, therefore, depends on the internet connection in order to function properly.

No external hardware is required in order to use the BaIn web application, as it can be used on any device that has an internet connection and a web browser installed.

3. External Interface Requirements

3.1 User Interfaces

In order to have access to the BaIn web application, a user needs to have an account. The user can be redirected to create account by clicking on the “Sign Up” button right under the "Welcome!" message displayed on the home page of the application.

To create an account, the user needs to enter their e-mail, first name, last name, date of birth, password and they also need to confirm their password. A refference code can be entered in order to have access to the profile of a child.

After the user has entered all the required information, they can click on the “Sign Up” button in order to create their account.

After the user has created their account, they can log in to the application by entering their e-mail and password by pressing "Log in!" on the home page of the application.

4. System Features

4.1 Authentification

4.1.1 Description and Priority

The authentification system is used to verify the identity of the user. The user needs to enter their e-mail and password in order to log in to the application. The user can also create an account by entering their e-mail, first name, last name, date of birth, password and they also need to confirm their password. A refference code can be entered in order to have access to the profile of a child.

The authentification system is used in the following situations:

  • The user wants to log in to the application.
  • The user wants to create an account.

The authentification system is a very important feature of the BaIn web application, as it is the only way to verify the identity of the user and to make sure that the user is the only one who has access to the information related to the children.

4.1.2 Stimulus/Response Sequence List

The following stimulus/response sequence list describes the authentification system:

  1. The user enters their e-mail, first name, last name, date of birth, password and they also need to confirm their password in order to create an account.
  2. The user enters a refference code in order to have access to the profile of a child.
  3. If all the information is correct, the user may continue by clicking on the “Sign Up” button.
  4. After all this, the user is redirected to the home page of the application.

If any of the information is incorrect, the user will be notified and an account will not be created.

4.1.3 Functional Requirements

The authentification system has the following functional requirements:

  • The user needs to enter their e-mail, first name, last name, date of birth, password and they also need to confirm their password in order to create an account.
  • The user needs to enter their e-mail and password in order to log in to the application.
  • The user needs to enter a refference code in order to have access to the profile of a child.

4.2 Adding multimedia

4.2.1 Description and Priority

The adding multimedia system is used to add multimedia to the profile of a child. The user can add a photo or a video to the profile of a child. The user can also add a description to the photo or video.

The adding multimedia system is used in the following situations:

  • The user wants to add a photo or a video to the profile of a child.
  • The user wants to add a description to the photo or video, or just information in the journal.

The adding multimedia system is a very important feature of the BaIn web application, as it allows the user to add multimedia to the profile of a child.

4.2.2 Stimulus/Response Sequence List

The following stimulus/response sequence list describes the adding multimedia system:

  1. The user clicks on the “Plus sign” button.
  2. The user selects a photo or a video from their computer.
  3. The user enters a description for the photo or video.
  4. The user may continue by clicking on the “Add” button.
  5. After all this, the user is redirected to the profile page of the child.

4.3 Exporting a CSV/JSON file

4.3.1 Description and Priority

The exporting a CSV/JSON file system is used to export a CSV/JSON file with all the information related to the child, including the photos and videos[*].

The exporting a CSV/JSON file system is a very important feature of the BaIn web application, as it allows the user to export a file containing all the information related to the child such that a backup can be made, should the BaIn web application be unavailable.

4.3.2 Stimulus/Response Sequence List

The following stimulus/response sequence list describes the exporting a CSV/JSON file system:

  1. The user clicks on the “Export CSV or JSON” button.
  2. The user selects the format of the file that they want to export.
  3. The user may continue by clicking on the “Export” button.
  4. After all this, the user is redirected to the profile page of the child, as the file is downloaded.

5.3 Security Requirements

When a user creates an account, the application will send an email to the email address that the user provided, in order to verify that the email address is valid. The user will have to click on a link in the email in order to verify the email address and activate the account. The user will not be able to log in to the application until the email address is verified.[*]

The data kept by the application regarding the children's profiles is not shared with any third party. The data is only used by the application and the users that have access to the profiles of the children.

Regarding the security of the application, the passwords of the users are stored in the database in an encrypted form, using the bcrypt algorithm. The passwords are not stored in plain text.
